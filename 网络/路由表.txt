一台电脑主机有路由表,路由器也有路由表,一般来说,一台主机电脑如果只有一个网卡的话,应该会有最少两条路由信息,
一条是公网路由,一条是局域网路由(它是不会经过路由器的路由表的,因为它走的是链路层,所以可以解析我们在局域网内传数据,速度是很快的)

在国产机上执行 route 命令，输出如下信息：
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
default         192.168.3.254   0.0.0.0         UG    425    0        0 br0
192.168.3.0     0.0.0.0         255.255.255.0   U     425    0        0 br0
对上面的解释：
Destination
    default : 当找不到匹配的路由时，走这条路由
    192.168.3.0 ：当掩码后的ip为192.168.3.0，则使用这条路由
Gateway
    192.168.3.254 ：默认的ip包，发到这个ip上（网关-路由器）
    0.0.0.0 ：没有网关，说明是在链路上（无需经过路由器，可直接到达目标主机）
Flags
    U (route is up) 路由正常
    H (target is a host) 主机路由
    G (use gateway) 使用网关的间接路由
    R (reinstate route for dynamic routing) 为动态选路恢复路由
    D (dynamically installed by daemon or redirect) 该路由由选路进程或重定向动态创建
    M (modified from routing daemon or rederict) 该路由已由选路进程或重定向修改
    ! (reject route) 阻塞路由

说明：接口部分，其实对应的是不同的网卡，因为电脑中有多个虚拟网卡，所以这里有多个接口

无论是在Windows下，还是在Linux下，都可通过 route 命令对路由表进行修改管理
从Windows的路由表可明显看出，路由表的作用就是根据ip，判断该数据包应该发往那个接口（网卡）
以及根据网关判断出之后网卡发给哪个网关或发往链路上的主机（mac地址）

路由表中，网关的作用：
    发送的链路层包，目的mac地址就是网关的地址
