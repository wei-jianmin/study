为什么不能用private：
private限定只能在当前类中使用，
这对于一个外部类而言，是讲不通的，
一个文件中可以定义多个类，
如果某个类加了private，
那这个类是属于哪个类的？哪个类能调用该私有类？
注意，private的限定范围是类级的，而不是文件级的，Java中没有文件级的相应限定关键字。

那类为什么要加public或frendly这样的关键字呢？
主要是因为Java中有包的概念，所以需要控制类是否要暴露给包外面，
而frendly对应的就是包这一级的，标明仅包内可用。
而public则标明包外（其他包下）可用。

那为什么不用用protect呢？
protect的可用范围比frendly更宽，除了包内可用外，包外的子类也可用，
那对于一个包中的类而言，说其对于包外的子类可见，是讲不通的。
